# Todo Tracking Guidelines

Whenever the user mentions that a topic, discussion, issue, or refactoring task should be:
- "tabled"
- "discussed later"
- "come back to later"
- "handled later"
- "put on a to-do list"

You MUST:
1. Immediately extract the relevant details of the issue, including location, description, and proposed solution.
2. Store or update this information in a persistent tracking file (e.g., `refactoring_todo.md` or `task.md` or a project TODO file in the workspace).
3. Do NOT implement or fix the issue immediately unless the user changes their mind and explicitly asks you to do so.
4. Present the items clearly to the user, confirming that they have been added to the tracking file.
